原文： 【Manus邀请码首测】深度实机评测全球首个通用AI Agent 🎮 Menus AI 发布与初期评价 - Menus AI 发布首日评价两极分化，既有“国运级AI”的赞誉，也有“套壳炒作”的负面评论。发布当晚，Open Menus 团队开源了他们的项目，邀请码被炒至几万元。 🛠️ Menus AI 功能与使用体验 - Menus AI 提供两种模式：标准模式（每天5次）和高投入模式（每天1次）。高投入模式生成质量更高，但沙盒崩溃可能导致高投入次数浪费。 - 用户通过分配任务并发布指令来使用 Menus AI。任务执行过程中会创建 todo list，并在完成后标记。 - Menus AI 使用了通益的蒸馏模型和 Cloud 3.5，生成报告、PPT 等文件。但大模型的幻觉问题仍未完全解决。 📊 Menus AI 的优缺点 - 优点：创新性强，性能表现优异，任务覆盖广，交付产物完整，可解释性强。 - 缺点：交互不友好，运行速度慢，任务执行过程中缺乏用户确认和动态修正，生成物质量参差不齐，单次任务训练成本高达2美元。 🔍 Open Menus 源码分析 - Open Menus 使用 Python 编程语言，包结构清晰，包括智能体、执行流程、提示词工具等。 - Agent 部分包含实现不同功能的 agent，如 Tool Agent 和通用 Menus Agent。Flow 管理 agent 执行流程，Promote 存储提示词模板，Po 包含调用工具。 - Open Menus 采用模块化设计，系统可扩展性高。Agent 使用 LM 进行推理，通过工具与外部环境交互，Planning Tool 处理复杂任务规划和执行跟踪。 🚨 Menus AI 的安全问题 - Menus 被爆出通过越狱方式获得虚拟机代码，执行后提供部署打包和沙箱运行时文件。
