原文： Claude Code Best Practices 🔧 Claude Code 简介 Claude Code 是 Anthropic 开发的命令行工具，作为研究项目旨在为工程师提供原生的 Claude 集成编码工作流。其设计理念是 低级别、无预设 ，提供接近原始模型的访问方式，同时兼顾灵活性、可定制性和安全性。虽然功能强大，但新用户可能需要一定学习曲线来掌握最佳实践。 📝 1. 自定义设置优化 a. 创建与使用 CLAUDE.md 文件 CLAUDE.md 是 Claude 自动加载的上下文文件，适合存放项目规范、常用命令、代码风格等信息。支持多位置放置： - 仓库根目录（推荐共享）或 CLAUDE.local.md （本地私有） - 父目录（适用于 monorepos）或子目录（按需加载） - 家目录 ~/.claude/CLAUDE.md （全局生效） 示例内容： # Bash 命令 - npm run build: 构建项目 - npm run typecheck: 类型检查 # 代码风格 - 使用 ES 模块语法（import/export）而非 CommonJS b. 工具权限管理 默认情况下，Claude Code 会请求敏感操作权限（如文件写入、bash 命令）。可通过以下方式自定义允许列表： - 会话中选择 &ldquo;Always allow&rdquo; - 使用 /permissions 命令添加工具（如 Edit 、 Bash(git commit:*) ） - 手动编辑 .claude/settings.json 或全局 ~/.claude.json - CLI 标志 --allowedTools 临时授权 c. 集成外部工具与 MCP GitHub CLI ( gh ) ：安装后可让 Claude 创建 PR、读取评论等。 MCP 服务器 ：作为客户端连接多个 MCP 工具（如 Puppeteer、Sentry），支持项目级或全局配置。 自定义斜杠命令 ：在 .claude/commands 目录存放 Markdown 模板，通过 / 调用（支持 $ARGUMENTS 参数）。 🚀 2. 高效工作流推荐 a. 探索-规划-编码-提交 让 Claude 读取相关文件/URL（明确要求不写代码） 使用 &ldquo;think&rdquo; 触发深度思考模式生成方案（如 think harder 分配更多计算资源） 实现代码并验证合理性 提交并创建 PR，更新文档 b. 测试驱动开发（TDD） 基于输入/输出编写测试（明确告知避免 mock 未实现功能） 运行测试确认失败 提交测试后，让 Claude 编写代码直至通过测试 c. 安全无监督模式（Safe YOLO） 使用 claude --dangerously-skip-permissions 绕过所有权限检查，适合修复 lint 错误或生成样板代码。 风险提示 ：建议在无网络容器中运行，参考 Docker 配置 。 d. 代码库问答与 Git 交互 新人上手 ：询问架构问题（如 &ldquo;日志系统如何工作？&rdquo;），Claude 会自动搜索代码库。 Git 操作 ：处理提交信息、历史查询、冲突解决等，支持 gh 命令简化 GitHub 交互。 💡 3. workflow 优化技巧 - 明确指令 ：避免模糊描述，例如将 &ldquo;add tests&rdquo; 改为 &ldquo;为 foo.py 编写用户登出边缘 case 的测试，不使用 mock&rdquo;。 - 提供视觉参考 ：粘贴截图、拖放图片或提供文件路径，辅助 UI 开发和图表优化。 - 多实例协作 ：同时运行多个 Claude 实例，分工编写/审查代码，或利用 git worktrees 隔离任务。 - Headless 模式 ：通过 claude -p "prompt" 集成到 CI/CD 流程，支持 JSON 输出自动化处理。
