原文： 斩获30.5k Star！Claude Code长期记忆插件Claude-Mem开源，实现跨会话上下文无缝保留 🔍 核心痛点与解决方案（背景） 开发者面临的关键问题 - 跨会话失忆严重 ：Claude Code每次新会话均&rdquo;从零开始&rdquo;，无法保留历史架构设计、API经验、代码规范等关键上下文。 - 低效的手动补偿方式 ：当前依赖手动维护CLAUDE.md文件、独立文档记录或重复说明项目背景，存在信息不完整、操作耗时、Token浪费等问题。 解决方案 ：Claude-Mem插件通过自动监听工具调用，将输出压缩为可搜索的语义化记忆，并在需要时智能提取相关上下文，实现跨会话知识连续性。该项目在GitHub上已获得 30.5k Stars 。 📌 项目核心机制与组件（技术架构） (一) 核心工作原理 Claude-Mem 通过自动捕获工具输出（1000～10000 Token），借助Claude Agent SDK压缩为约500 Token的语义化观测记录，按类型分类（决策、Bug修复、功能、重构、发现、变更）并打标签后，存入本地SQLite数据库。在后续会话中自动注入相关上下文，实现知识持久化。 (二) 六大核心组件 组件名称 功能描述 插件钩子 6个生命周期钩子，覆盖会话全流程： ① context-hook.js（SessionStart）：启动Bun Worker并注入上下文 ② user-message-hook.js（UserMessage）：开发调试 ③ new-hook.js（UserPromptSubmit）：创建新会话并保存提示词 ④ save-hook.js（PostToolUse）：捕获工具执行记录 ⑤ summary-hook.js（Stop）：生成会话摘要 ⑥ cleanup-hook.js（SessionEnd）：标记结束并清理 智能安装 带缓存的依赖检查器，含预钩子脚本（在context-hook前运行） Worker服务 通过Claude Agent SDK处理数据压缩，暴露HTTP API（默认37777端口，10个搜索端点） 数据库层 基于SQLite、FTS5和ChromaDB存储会话和观察记录 mem-search Skill 基于技能的搜索，支持渐进式披露（v5.4.0+版本） 查看器UI 基于Web的实时记忆流可视化界面 📋 使用指南（操作说明） (一) 环境要求 运行环境 ：Node.js 18.0.0+、支持插件的最新版Claude Code 依赖 ：Bun（JavaScript运行时和进程管理器，缺失会自动安装） 存储 ：内置SQLite 3 系统支持 ：Windows、macOS、Linux (二) 安装方式 安装类型 操作步骤 快速安装 1. 终端启动Claude Code新会话 2. 输入命令： /plugin marketplace add thedotmack/claude-mem/plugin install claude-mem 3. 重启Claude Code，自动加载历史上下文 源码安装 1. 克隆仓库： git clone https://github.com/thedotmack/claude-mem.git 2. 进入目录： cd claude-mem 3. 安装依赖： npm install 4. 构建项目： npm run build 5. 启动服务： npm run worker:start 💡 核心价值总结 Claude-Mem通过为Claude Code提供&rdquo;长期记忆&rdquo;能力，解决了跨会话上下文丢失问题，避免重复建立上下文、技术决策遗忘等痛点，显著提升开发者与AI编程助手的协作效率，优化工作流程连续性。 📎 项目信息 项目地址 ： https://github.com/thedotmack/claude-mem
