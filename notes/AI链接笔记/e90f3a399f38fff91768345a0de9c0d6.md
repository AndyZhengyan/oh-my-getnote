原文： microsoft/ai-agents-for-beginners 🎓 课程概述 - “AI Agents for Beginners”是一门全面的教育课程，旨在教授构建AI代理的基本概念和实践方法。课程从基础代理概念到高级模式，最后考虑生产应用，提供了结构化的学习路径。 - 课程聚焦于微软的三个主要AI代理框架：Semantic Kernel、AutoGen和Azure AI Agent Service，并通过Python代码示例展示实现方法。 📚 课程目的与结构 - 课程目标是通过10节课，为初学者提供理解和构建AI代理所需的基本知识。 - 课程包含实用代码示例、关键代理设计模式探索，涵盖多个框架和集成方法，逻辑清晰，帮助学习者理解理论概念和实际应用。 🧠 AI代理基础 - AI代理是使大语言模型（LLMs）通过工具和知识扩展能力来执行操作的系统。其关键组件包括环境、传感器、执行器、LLM和记忆/知识。 - 课程涵盖多种类型的AI代理，如简单反射、基于模型的反射、基于目标、基于效用、学习型、分层和多代理等，并给出了相应的示例用例。 💻 AI代理框架 - Semantic Kernel ：企业级SDK，用于通过内核函数、插件架构和简化集成，将AI功能集成到应用程序中。 - AutoGen ：专注于多代理系统的研究框架，使用参与者模型进行代理通信，提供事件驱动架构。 - Azure AI Agent Service ：托管云服务，提供与Azure安全和合规功能的集成、AI模型市场访问、标准化工具定义和企业级部署能力。 🎨 设计模式 - 工具使用模式 ：使代理能够与外部工具和服务交互，提供函数调用、API集成和代码执行功能。 - Agentic RAG ：通过查询优化、上下文检索和自我纠正机制，提高代理的上下文感知能力。 - 规划模式 ：帮助代理分解复杂任务，将目标分解为步骤，优化步骤顺序，并处理错误和重新规划。 - 多代理模式 ：创建专门代理系统，通过定义协议进行通信，交接任务，协作解决复杂问题。 - 自我反思和适应模式 ：通过监控性能和结果，调整策略，从过去的交互中学习。 🛠️ 实现技术 - 语言模型 ：使用OpenAI / Azure OpenAI模型和GitHub Models。 - 数据存储解决方案 ：包括Azure AI Search、ChromaDB和SQL数据库。 - 用户界面选项 ：使用Chainlit进行基于聊天的界面，使用HTTP API与其他应用程序集成。 📋 适用场景 - AI代理特别适用于开放式问题、多步骤流程和需要通过反馈和适应不断改进的系统。 📦 课程资源 - 课程提供代码示例、视频教程、多语言支持和社区支持。 🔧 开发环境搭建 - 先决条件 ：需要Python 3.12或更高版本、GitHub账户、Azure账户和Azure AI Foundry账户。 - 开发环境选项 ：可以选择本地Python环境或开发容器（GitHub Codespaces或VS Code with Docker）。 - 所需依赖 ：包括AutoGen、Semantic Kernel、Azure AI等框架库，以及chromadb、chainlit等附加库。 - 配置设置 ：需要设置环境变量，包括GitHub令牌和Azure配置，并使用Microsoft Entra ID进行Azure身份验证。 - 验证设置 ：完成设置后，需要测试环境变量加载、Azure连接，并运行简单示例。 - 故障排除 ：提供了常见问题的解决方案，如缺少依赖、环境变量问题、身份验证错误等。
