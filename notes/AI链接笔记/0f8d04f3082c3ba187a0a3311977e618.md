原文： 从“人驱动”到“模型驱动”:聊聊 Agent 在 2025 年的爆发与挑战 🌟 Agent 技术发展与思考转变 - 随着人工智能等技术发展，Agent 技术从理论走向应用。作者所在团队负责的 AI Coding 产品 Aone Copilot 在阿里广泛使用，但作者曾因模型能力和业务逻辑复杂性，对大模型解决复杂问题持怀疑态度，迟迟未做 Agent 产品。 - 作者曾认为 Agent 会经历局部自动化、广泛自动化和极端自动化三个模式，集团内已有相关应用并遇到问题。如今随着多模态和 Reasoning 模型能力强化，作者认为到了从“人驱动”到“模型驱动”的关键节点。 💪 模型技术进步推动产品进步 - 推理能力进步 ：去年下半年，以 O1 和 DeepSeek 为代表的模型在 Reasoning 模型上进步显著，打消了大家对模型推理能力的疑虑。模型能更理解用户需求，无需在 Prompt 设计上大费周章，在需求理解层面达人类水准，还能纠正错误问题。 - 多模态模型能力进步 ：过去处理文档图片困难，如今大模型理解图片能力提升，能抓出复杂架构图重要信息。可将图片与文字整合处理，用于检索增强生成和生成摘要。在构建答疑系统时，能通过分析浏览器页面截图理解用户问题，改善业务系统智能化。 - 代码能力进步 ：模型在图片复刻和输出长篇内容上表现出色，能 1 比 1 复刻图片，一次性生成上千行前端代码且无语法问题，提升创意从 0 到 1 的效率。 - 自我认知能力进步 ：模型能更理解自身局限性，如 Claude 3.7 面对复杂数字计算，能认识到无法计算并给出正确 Python 脚本，避免错误累积。 🚀 产品进步推动理论进步 - Devin 开创通用 Agent 时代 ：Devin 是首个具体化且功能广泛化的 Agent 产品，引领 Agent 领域变革。它能自主规划任务、反思、使用工具、联网等，解决了长上下文记忆压缩与提取、规划与重新规划、自我总结与学习反思等问题。虽订阅费用高，但后续产品多借鉴其理念，其模式被称为“Planning Pattern”，也叫“异步 Agent”“通用 Agent”，有望释放人类“十倍”潜能。 - Cursor 和 Cline 开创本地 IDE 上的 Agent 模式 ：Cursor 和 WindSuf 在本地 IDE 中有创新，其 Agent 模式能根据用户需求修改代码、执行命令。Cursor 解决了用户信任问题，采用“Reflection Pattern”，是“同步 Agent”。本地 IDE 中 Agent 效果取决于模型能力和搜索功能，可利用本地资源处理工程上下文问题，降低代码修复成本。 🔍 不同场景下 Agent 模式差异 - 通用 Agent 需独立云端运行环境，支持异步交付，对延迟要求不高，工具使用不受限，但要确保交付质量，执行效率可能降低。 - 本地 Agent 部署于本地 IDE，重视执行效率，与人类操作同步，任务相对简化，受 PC 设备限制，安装新工具可能引发隐私和安全担忧。 😫 通用 Agent 面临的挑战 - 工程挑战 - 大脑构建 ：通用 Agent 的“大脑”要具备任务规划与执行、反思和重新规划、识别和选择工具、知识引入和学习、并行处理子任务等能力，还可内置 IDE 功能。 - 评估难题 ：现有以 Query 和 Answer 为核心的评测集无法全面评估通用 Agent 综合能力，评估过程因执行动态变化而困难，可能需引入人工评估和构建突发场景。 - 记忆问题 ：Agent 记忆分短期和长期，短期记忆需信息压缩，但可能忽略关键细节；长期记忆通过经验存储实现，但固化经验可能限制模型能力，如何甄别和利用经验是挑战。 - 模型挑战 - 指令跟随能力不足 ：除 Claude 系列外，其他模型难以遵循复杂指令。 - 长上下文能力弱 ：噪音信息增多时，模型找关键信息和理解能力变弱。 - 推理规划和反思能力欠缺 ：推理与规划能力是解决复杂问题关键，Agent 还存在难以从错误长轨迹恢复和陷入局部循环的问题。 - 开源模型进步存疑 ：以往模型优化方式在 Agent 场景不一定适用，Claude 优势明显，国内模型在提升相关能力上毫无头绪。 🤔 通用 Agent 未来走向 - 通用 Agent 不会被模型取代，与模型是共生关系，可帮助模型应对环境感知等问题，还可能产生规模效应。但以工作流为核心的专业 Agent 可能被淘汰。 📢 结尾呼吁与会议推荐 - 作者认为 Agent 是划时代的范式革命，邀请对重构人机关系底层逻辑好奇的人加入，提供了联系方式和招聘岗位。 - 推荐 4 月 10 - 12 日召开的 QCon 全球软件开发大会（北京站），作者将分享阿里在智能研发中应用大模型的经验。
