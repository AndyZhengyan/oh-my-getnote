原文： Andrej Karpathy 谈 AI 时代软件范式变革与语言模型应用 🎙️ 演讲背景与核心观点 - 6 月 16 日在旧金山 YC AI Startup School 上，Andrej Karpathy 发表演讲，指出软件正进入快速重写与重构时代，大型语言模型是变革核心动力。 - 现在是进入软件行业的绝佳时机，过去 70 年软件基本范式变化小，近几年却有两次底层变革。 💻 软件三代演进 - 软件 1.0：用代码编程计算机，开发者手写代码。 - 软件 2.0：用权重参数“编程”神经网络，核心是神经网络权重，通过数据训练模型，Hugging Face 类似这一时代的 GitHub。 - 软件 3.0：用英文 prompt 编程语言模型，prompt 成程序，语言模型是通用执行器，降低软件交互门槛，颠覆传统开发流程。当下三种范式并行，工程师需灵活切换工具。 🧠 如何理解大模型 - 语言模型类似“操作系统”，有闭源与开源之分，底层结构类操作系统，以云端部署为主，是“时间共享计算”系统。 - 个人智能未真正到来，语言模型技术扩散路径逆转，大众是首批用户。 🤖 LLM 心理机制 - 语言模型是“人的精神的随机模拟体”，有双重特性，既是“数字雨人”又是初学者，存在“顺行性遗忘症”和安全隐患。 - 其运行机制像每天重启的同事，需外部管理“工作记忆”，使用时要引导能力、规避短板。 💡 Karpathy 看到的机会 - 部分自治型应用 ：如 Cursor 等，自动管理上下文、编排多模型调用，有图形界面，用户可通过“自主性滑块”控制 LLM 干预程度。 - vibe coding ：基于自然语言与智能系统协作的全新编程方式，降低开发门槛，唤起数十亿人创造冲动。 - 产品构建实践 ：Karpathy 用 vibe coding 一天构建 iOS 应用，开发 MenuGen 应用，反思 GUI 与流程逻辑应适配 Agent。 - 面向 Agent 的软件系统 ：应构建为 Agent 而生的软件系统，改造文档结构与内容表达方式，建立面向 LLM 的中介转接层。
