原文： 智能体构建指南：开启工作流程自动化新时代 🤖 智能体概述 - 智能体以强大自主性和适应性引领自动化新潮流，能独立完成任务、处理复杂流程、在模糊动态环境决策，与传统软件不同。 - 核心特征包括自主性（用户授权下独立执行任务）、适应性（根据环境调整行为）、工具使用能力（调用外部工具和 API）、决策能力（基于大模型推理做决策），如客户服务场景中自动处理咨询。 ❓ 构建时机 - 适合传统确定性和基于规则方法无法胜任的场景，如复杂决策（客户服务退款审批）、难以维护规则（供应商安全审查）、严重依赖非结构化数据（家庭保险索赔处理）。 - 构建前要验证用例是否满足标准，避免盲目追求 AI 而舍弃原系统稳健性。 🧠 设计基础 - 核心组件：模型（提供推理和决策支持的大模型）、工具（外部函数或 API）、指令（指导行为的指南和安全防护措施）。 - 选择模型：设置评估建基线，用最佳模型达准确度目标，用小模型优化成本和延迟。 - 定义工具：通过 API 扩展功能，分数据、行动、编排三类，工具要有标准化定义，如查询数据库、发送邮件、退款智能体等。 - 配置指令：高质量指令至关重要，编写时可使用现有文档、提示分解任务、定义明确行动、捕获边缘情况，还可用高级模型自动生成。 🔄 编排模式 - 单智能体系统：逐步添加工具处理任务，保持复杂性可控，通过循环运行，有常见退出条件，可使用提示模板简化维护。 - 多智能体系统：分为经理模式（中央经理智能体协调专用智能体）和分散模式（智能体相互交接任务）。经理模式适合单智能体控制工作流程，分散模式适用于专用智能体接管任务场景。 🛡️ 安全防护措施 - 是基于大模型部署的关键，结合多种认证和安全措施，像分层防御机制。 - 类型包括相关性分类器、安全分类器、PII 过滤器、审核、工具安全措施、基于规则的保护、输出验证。 - 构建时关注数据隐私和内容安全，根据实际情况添加和调整，在 Agent SDK 中可设置。 👨‍💼 人工干预 - 是关键保障，在部署初期重要，能识别失败和边缘情况。 - 触发情况有超过失败阈值和高风险行动，如多次尝试后无法理解客户意图、取消用户订单等。 🎉 总结 - 智能体是工作流程自动化新方式，适合复杂决策等用例。 - 构建要从坚实基础开始，选择合适编排模式，重视安全防护和人工干预。 - 成功部署要从小处着手，逐步增强能力，实现商业价值。
