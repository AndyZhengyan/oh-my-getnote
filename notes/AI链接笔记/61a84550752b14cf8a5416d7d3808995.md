原文： 从 OpenAI Harness Engineering 蒸馏出四个 Skill，Agent 跑了 25 小时 📌 背景与核心动机 文章来源 ：OpenAI于2026年2月11日发布的《Harness engineering: leveraging Codex in an agent-first world》，作者Ryan Lopopolo。 核心数据 ：3名工程师，5个月内完成100万行代码（零手写），人均日产出3.5个PR。 个人痛点 ：Agent执行长任务时存在三大问题——Context重置后状态丢失、任务中途终止、模糊点等待人工确认。 核心目标 ：将OpenAI的云端Codex+自研基础设施方案，蒸馏为适用于本地环境的Claude Code Skill。 🔍 内容蒸馏方法论 采用三级分类法处理原文内容： 类别 定义 处理方式 直接可用的机制 实现逻辑清晰，可完全照搬 直接集成 思路可借鉴但实现需改 核心思想有价值，但需适配本地场景 改造后落地 暂不考虑的方案 与当前业务场景冲突或复杂度较高 暂不采纳 🛠️ 关键机制蒸馏与落地 (一) 直接可用的机制 进度文件即上下文 OpenAI方案：将execution plans和decision logs存储于仓库，Context重置后通过读取文件恢复状态。 落地实现： harness 模块中的 harness-tasks.json + harness-progress.txt + git log 组合，10秒内恢复会话。 精简AGENTS.md设计 OpenAI经验：早期&rdquo;大而全&rdquo;的AGENTS.md因文件过大、规则冗余、信息过期等问题失败，后改为100行精简版作为导航地图，详细知识存储于 docs/ 目录。 落地实现：直接应用于 harness 的知识架构设计。 结构测试机械化约束 OpenAI方案：自定义linter+结构测试强制执行分层约束，错误信息包含修复指引。 落地实现：成为 architecture-guardrails 的核心机制。 (二) 思路借鉴与改造实现 可观测性适配 OpenAI方案：每个git worktree配备独立Victoria Logs+Metrics+Traces，Agent通过LogQL/PromQL/TraceQL查询。 落地改造：针对业务流测试场景，设计&rdquo;证据包&rdquo;机制（verdict.json+请求响应记录+数据库快照+过滤日志），判定依据业务不变式而非脚本退出码，形成 closed-loop-testing 核心。 Entropy管理优化 OpenAI方案：通过golden principles+后台Agent定期扫描修复&rdquo;AI slop&rdquo;（模式漂移）。 落地改造：暂不实现后台扫描，改为将golden principles编码为结构测试集成于CI，结合 architecture-guardrails 的ratchet策略（遗留违规加入allowlist，新代码必须通过）。 (三) 暂不考虑的方案 放松合并门禁 ：OpenAI内部工具采用最小阻塞合并策略，测试波动通过后续运行解决；但因业务涉及支付、订单，需严格阻塞flaky test，故 closed-loop-testing 明确区分blocking check。 Agent-to-Agent review ：OpenAI实现&rdquo;人类可选review&rdquo;，主要由Agent完成；本地采用Codex开发+Claude Code review的跨模型组合，利用模型训练差异互补盲区。 💡 四大核心Skill设计 蒸馏后形成问题分层解决方案： Skill名称 解决问题 核心功能 harness Agent持续运行与状态恢复 执行引擎，支持断点续跑 closed-loop-testing 业务流测试验证 分阶段（V1/V2/V3）产出证据包 architecture-guardrails 架构规则机械化执行 结构测试+ratchet策略 harness-marathon 长时运行策略 分析Agent停止原因（工作耗尽/卡住/Context退化）并优化 📊 实践效果与关键发现 核心指标 ：单次最长运行25小时（Go服务重构，40+任务），任务一次通过率80%（20%中50%为设计分歧，50%为Prompt优化空间）。 关键发现 ： 1. 进度文件价值 ： harness-progress.txt 记录完整操作日志，解决session挂起后的状态恢复问题，避免重复理解项目状态。 2. 马拉松三定律 ：&rdquo;Zero decision points&rdquo;（提前写入所有外部依赖）最易被忽略，漏写依赖（测试数据路径、API key等）是主要停顿原因。 3. Doom Loop检测 ：文件编辑超6次报警、12次强提醒机制，有效识别Agent无效循环。 4. 分阶段测试优势 ：V1（内部路径）成功率近100%，V2（外部回调）支持replay模式，提升整体稳定性。 5. 跨模型协作 ：Codex开发+Claude Code review组合，利用模型盲区互补提升PR质量。 🎯 蒸馏思路本质 提炼技术文章为实用工具的核心三问： 1. 解决的本质问题 ：关注&rdquo;为什么做&rdquo;而非&rdquo;做了什么&rdquo;，如进度文件本质是解决&rdquo;Context重置后状态恢复&rdquo;。 2. 场景适配性 ：判断问题在本地场景是否存在及表现形式，如可观测性从性能调优转为业务流验证。 3. 简化实现 ：优先采用轻量方案，如用结构测试+人工检查替代OpenAI的doc-gardening Agent。
