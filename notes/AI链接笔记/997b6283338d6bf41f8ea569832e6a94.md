原文： DAPO技术报告解读：SFT、RL及损失计算分析 📖 DAPO技术亮点 - seed&清华发表的DAPO报告，将32b - base做到了aime50分的效果，很值得参考。 💡 SFT与RL流程 - sft可看作rl的特例，rl能更好利用负样本。DAPO核心是dynamic - sampling，即根据模型在prompt的表现动态决定采样预算，难的prompt多采样，简单的少采样，还会过滤难或简单的prompt。 - sft阶段常用类似策略做拒绝采样、筛选多样性response。一个良好的online - dynamic - sampling流程远胜不靠谱的算法trick。 - 做好sft后，将其流程在线化，应用到replay - buffer的数据构造中，配合挑选/打分/筛选策略，可将sft优秀流程迁移到online - rl，同时要让replay - buffer和主代码解耦。 - 若sft流程难在线化运行，会导致做sft的人力重复劳动，做rl的重复sft数据流程造成资源浪费，做agent - rl时若sft没积累agent环境数据，环境稳定性会影响rl训练。 🧐 Token - level - loss分析 - DAPO提到的token - level - loss在24年末社区引发讨论，梯度累加较大时，梯度累加训练和大batch训练loss差异大。主流框架实现的loss计算比大batch计算的loss更大，对长文本训练不利。 - openrlhf/verl中，micro - batch - loss为token - level - loss计算，有梯度累加时也有类似问题，前期loss过大，优化激进。 - 实现梯度累加内的token - level - loss，计算loss时按各维度求和再除以当前ga内的总token数。 - ga - token - level - loss会让grpo优化更稳定，无token - level - loss训练到后期，repeatness、format崩溃会显著上升。 ⚖️ reinforce_baseline和grpo对比 - reinforce_baseline的advantage计算是r - group_mean + 全局归一化，grpo的advantage计算是(r - group_mean)/group_std（group归一化）。 - 当reward = 0/1时，reinforce_baseline和grpo只差一个系数。全局std大于group_std，当采样样本无穷多时，若不对group样本调整，group_std标准差小到一定程度会让loss急剧增加。 - 二者有类似的training - dynamics，grpo收敛速度好于reinforce_baseline，但稳定性不如，全局样本方差较小时，reinforce_baseline崩溃得晚一些。 📋 总结 - 把sft流程搬到online - replay - buffer采样流程中，配合稳定的online环境和鲁棒的rl方法，就能做好online - rl。 - token - level - loss在ga层面实现很重要，rl训练时梯度累加较大，可一次采样多次参数更新更offpolicy些。 - reinforce_baseline更稳定，grpo前期优化较猛。
