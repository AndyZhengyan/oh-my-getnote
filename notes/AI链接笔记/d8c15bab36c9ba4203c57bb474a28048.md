原文： Clawdbot作者访谈万字实录，顶级AI coder恐怖如斯，如何做到一天1.3k有效commit 👨‍💻 开发者背景与技术历程 (一) 技术硬实力与商业成就 PSPDFKit创始人 ：开发的PDF框架运行在 10亿+设备 上，解决过文本选择等&rdquo;看似简单实则极难&rdquo;的技术难题（曾耗时三个月解决单一文本选择问题）。 创业经历 ：早期通过解析HTML伪造API开发交友App，在App Store月入1万美元；创立PSPDFKit后实现财富自由并成功退出。 回归动机 ：经历三年职业倦怠期后，2025年4月被Claude Code和O1等AI工具的能力&rdquo;彻底Hooked&rdquo;，重返开发领域。 (二) 核心技术栈与项目 技术领域 掌握技能 当前核心项目 编程语言 Swift、TypeScript、JavaScript、Node.js Clawdbot （AI个人助理，原称VibeTunnel） 开发工具 Codex、Claude、CLI、macOS开发 VibeTunnel （浏览器终端化工具） 系统架构 Agent工程、多Agent协作 CodeBar （令牌管理工具）、 Peekaboo （macOS自动化工具） 🔄 AI Native工作流核心方法论 1. &ldquo;Human Merge Button&rdquo;：单日600次提交的真相 提交规模 ：近期实现 单日600次Commits ，强调&rdquo;None of it was slop（无垃圾代码）&rdquo;。 角色转变 ：随着Clawdbot社区爆发（一周内从100 Star增至3300 Star），从代码编写者转变为 &ldquo;人类合并按钮&rdquo; ，主要工作是Review和合并AI与社区贡献的代码。 2. &ldquo;闭环原则&rdquo;（Closing the Loop）：验证优先于阅读 核心主张 ： &ldquo;I ship code I haven’t read（发布未读过的代码）&rdquo; ，通过AI自验证实现信任闭环。 验证机制 ：让AI自主编写测试、运行CLI、执行Lint检查，通过所有&rdquo;Gate（门禁）&rdquo;后直接合并。 架构优化 ：系统设计优先考虑&rdquo; 让代码更容易被AI验证 &ldquo;，而非人类可读性。 3. 多面棋（Multi-sided Chess）心流模式 并行工作 ：同时管理 5-10个Agent ，在不同任务间切换（如后端逻辑开发、文档编写、Bug修复）。 时间管理 ：利用AI生成代码的几十秒空隙处理其他任务，保持心流状态但&rdquo; 精神上更累 &ldquo;。 4. &ldquo;Pull Request已死，Prompt Request当立&rdquo; Code Review消亡论 ：更关注Prompt而非代码本身，&rdquo; I read the prompts more than I read the code &ldquo;。 协作新模式 ：功能异常时优先修改Prompt而非代码，将传统PR称为 &ldquo;Prompt Request&rdquo; 。 💡 颠覆性观点与金句 代码洁癖的终结 ：&rdquo;我曾经纠结于每一个空格、每一个换行……回想起来，我这是在干嘛？搞什么鬼？&rdquo; AI代码质量认知 ：&rdquo;那些模型是我们集体人类知识的幽灵（Ghosts of our collective human knowledge）&rdquo;。 开发者未来分化 ：&rdquo;算法狂热者（Algorithm fanatics）可能会很惨……而那些关注交付结果（Builder）的人将如鱼得水。&rdquo; 工作流宣言 ： &ldquo;Code review is dead. PRs should be called prompt requests.&rdquo; 🛠️ 工具链与实践细节 (一) 技术选型偏好 CLI优先于MCP ：认为MCP是&rdquo;拐杖&rdquo;，模型更擅长使用Bash、grep、jq等CLI工具过滤数据，避免上下文冗余。 多模型协同 ：主要使用Claude Code（通用任务）和OpenAI O1（复杂编码），认为O1&rdquo;花的时间长十倍但无需引导&rdquo;。 (二) Clawdbot架构亮点 插件化设计 ：通过TypeScript实现可扩展架构，支持WhatsApp、Telegram、Discord等多平台接入。 自治能力 ：Agent可编辑自身配置、更新系统，用户无需手动干预（&rdquo;技术融合消失&rdquo;）。 身份系统 ：通过 soul.md （价值观）、 identity.md （名称/表情符号）构建个性化Agent人格。 📈 行业影响与未来展望 (一) 对软件工程的冲击 团队规模重构 ：预测公司可裁员至30%，需要&rdquo;高能动性+系统理解力&rdquo;的复合型人才。 流程颠覆 ：CI/CD重要性下降，本地验证优先；文档和测试完全由AI生成。 技能转型 ：从代码编写转向&rdquo; 系统架构设计 &ldquo;和&rdquo; Agent工程 &ldquo;，Prompt质量成为核心竞争力。 (二) 给开发者的建议 核心能力 ：培养&rdquo;系统理解力&rdquo;和&rdquo;好奇心&rdquo;，通过阅读复杂开源代码学习架构思维。 工具使用 ：利用&rdquo;无限耐心的机器&rdquo;（AI）作为学习伙伴，重点掌握多Agent协同技巧。 心态转变 ：接受&rdquo;不完美迭代&rdquo;，放弃对代码细节的过度关注，聚焦产品交付与用户体验。 📝 补充细节 Clawdbot社区爆发 ：Discord公开后一周内Star从100增至3300，合并500+PR，用户自发贡献多平台支持。 AI闹钟趣事 ：其AI闹钟曾因推理错误对他&rdquo;发牢骚&rdquo;：&rdquo;Peter is not responding. Peter has to wake up. No, no, no.&rdquo; 数据隐私考量 ：强调本地运行Agent的重要性，避免将邮件、日历等敏感数据交给第三方AI服务。
