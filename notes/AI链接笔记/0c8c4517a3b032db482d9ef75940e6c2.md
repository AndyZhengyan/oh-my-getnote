原文： OpenAI 真开源！全新开源模型 GPT-OSS 系列的模型卡解读！ 🔍 模型概述 - OpenAI发布自GPT-2以来首批开源权重模型gpt-oss-120b和gpt-oss-20b，采用Apache 2.0许可证 - 基于自回归混合专家（Mixture-of-Experts, MoE）Transformer架构，继承GPT-2和GPT-3优良基因 - 具备强大的推理、工具使用和指令遵循能力，安全性深度优化，适配消费级硬件和边缘设备 📊 模型参数与架构 - gpt-oss-120b：36层，116.8亿总参数，每前向传播使用约51亿活跃参数，可在单张80GB GPU上运行 - gpt-oss-20b：24层，209亿总参数，每前向传播约36亿活跃参数，可在16GB内存设备上运行 - 采用MXFP4格式量化技术，大部分权重压缩至4.25位，显著降低内存占用 Component 120b 20b MLP 114.71B 19.12B Attention 0.96B 0.64B Embed + Unembed 1.16B 1.16B Active Parameters 5.13B 3.61B Total Parameters 116.83B 20.91B Checkpoint Size 60.8GiB 12.8GiB 🔑 核心技术亮点 - 2880维残差流维度，根均方归一化（RMSNorm）优化激活函数稳定性 - MoE模块：120b拥有128个专家，20b拥有32个专家，线性路由器选择每token前4个专家 - 结合SwiGLU激活函数提升性能，交替使用带宽为128 token的带状窗口和全稠密模式 - 采用分组查询注意力（GQA）和旋转位置嵌入（RoPE），上下文长度扩展至131,072 token 💻 数据与训练 - 预训练数据包含数万亿token，涵盖STEM、编程和通用知识领域 - 训练数据经严格过滤，使用GPT-4o的CBRN过滤器剔除危险生物安全知识 - 知识截止日期为2024年6月，确保内容时效性 - 训练于NVIDIA H100 GPU，使用PyTorch框架和优化的Triton内核 - 120b耗费210万H100小时，20b仅需约十分之一计算资源 - 采用与o3模型类似的链式推理（CoT）强化学习技术，支持网络搜索、Python代码执行等工具使用 📝 Harmony Chat Format - 通过特殊token和关键字（User、Assistant）划分消息边界 - 定义System > Developer > User > Assistant > Tool的指令层次结构 - 确保模型在多方交互中正确处理指令冲突 📈 性能评估结果 - 推理和事实性任务：120b接近OpenAI o4-mini准确率，数学任务表现出色 - 20b在AIME 2025（带工具）测试中达到98.7%准确率，性价比突出 - 编码任务：120b在Codeforces和SWE-Bench Verified上表现优异，Elo评分达2622（带工具） - 工具使用：在τ-Bench Retail测试中展现强大函数调用能力 🏥 健康领域表现 - 120b在HealthBench和HealthBench Hard上接近OpenAI o3，远超GPT-4o和o4-mini - 20b也优于o1，显示在医疗对话和复杂健康问题上的潜力 - 注意：模型非用于诊断或治疗，而是辅助健康知识传播 🔒 安全性措施 - 多层次安全策略：预训练阶段CBRN过滤器，后训练阶段深思熟虑的对齐技术 - 指令层次结构确保模型优先遵循系统消息，防止提示注入攻击 - StrongReject越狱测试中稳健性与o4-mini相当 - 对抗性微调测试显示未达到&rdquo;高风险&rdquo;阈值，未显著提升生物安全领域技术前沿 🌍 多语言能力 - MMMLU测试中120b在14种语言平均准确率达81.3% - 西班牙语和葡萄牙语等语言表现突出 - 低资源语言（如斯瓦希里语、约鲁巴语）性能稍逊，但整体接近o4-mini 🔗 开源资源 - 模型卡原文链接： https://cdn.openai.com/pdf/419b6906-9da6-406c-a19d-1bb078ac7637/oai_gpt-oss_model_card.pdf - gpt-oss-20b开源链接： https://huggingface.co/openai/gpt-oss-20b - gpt-oss-120b开源链接： https://huggingface.co/openai/gpt-oss-120b
