原文： 火遍全网的Clawdbot究竟是如何记住一切的？ 深度解析记忆机制 🦞 项目概况与核心功能 项目背景 - 开发者 ：Peter Steinberger - 当前名称 ： Moltbot （原Clawdbot，因与Claude读音近似更名） - GitHub数据 ：截至2026年1月30日， 106,000+星标 ，开源项目 核心功能亮点 1. 本地化运行 ：个人设备直接运行，访问本地文件与应用 2. 跨平台通信 ：集成WhatsApp、Telegram、Discord等主流社交APP 3. 主动交互能力 ：通过定时任务自主发起操作 4. 自我迭代机制 ：支持Skills和MCP（模块化能力平台）扩展 5. 长期记忆系统 ：全天候上下文保持，完整记录用户交互历史 🧠 记忆机制深度解析 1. 记忆与上下文的本质区别 维度 记忆 (Memory) 上下文 (Context) 存储形式 磁盘文件（MEMORY.md + memory/*.md + 会话记录） 动态临时数据（系统提示词+对话历史+工具结果） 持久性 重启/数月后依然存在 仅存在于单次请求中 容量限制 无限增长 受模型上下文窗口限制（如200k token） 成本特性 存储成本低，无API费用 每个token影响API成本和速度 检索能力 支持语义搜索（索引化） 实时处理，无持久检索能力 2. 双层记忆架构 文件结构 ~/clawd/ ├── MEMORY.md - 第二层：长期记忆（提炼的关键事实） └── memory/ ├── 2026-01-26.md - 第一层：当日日志（实时记录） ├── 2026-01-25.md - 历史日志 └── ... 第一层：当日日志（ memory/YYYY-MM-DD.md ） - 特性 ：类似&rdquo;日记本&rdquo;，记录实时讨论、操作和临时信息 - 示例 ： # 2026-01-26 ## 10:30 AM - API讨论 与用户讨论REST和GraphQL. 决定：为简化操作，采用REST Key endpoints: /users, /auth, /projects. 第二层：长期记忆（ MEMORY.md ） - 特性 ：AI自动提炼的结构化知识库，包含用户偏好、重要决策等 - 示例 ： # 长期记忆 ## 用户偏好 - 相对于JavaScript，更喜欢TypeScript - 喜欢精简的解释 ## 重要决定 - 2026-01-15: 数据库选择了PostgreSQL - 2026-01-26: 使用Tailwind CSS进行样式设计 3. 记忆检索与写入机制 检索策略 - 双引擎并行搜索 ： - 向量搜索 （语义相似度，权重70%） - BM25关键词搜索 （精确匹配ID/日期/名称，权重30%） - 评分公式 ： 总分 = (0.7 * vectorScore) + (0.3 * textScore) - 阈值过滤 ：默认最低评分0.35，可自定义配置 核心检索工具 1. memory_search ：跨文件语义搜索 { "name": "memory_search", "parameters": { "query": "我们对API做了什么决定?", "maxResults": 6, "minScore": 0.35 } } memory_get ：精准读取文件指定行内容 json { "name": "memory_get", "parameters": { "path": "memory/2026-01-20.md", "from": 45, "lines": 15 } } 写入规则 （由AGENTS.md提示词驱动） | 触发场景 | 存储目的地 | 内容类型 | | :&mdash;&mdash;&mdash;&ndash; Store durable memories now&rdquo; #### 3. 修剪 (Pruning) - 目标 ：处理工具调用产生的庞大输出（如50,000字符日志） - 策略 ： - 软修剪 ：长文本截断为首尾各1500字符（共4000字符） - 硬清除 ：旧工具结果标记为&rdquo;[Old tool result content cleared]&rdquo; - 特性 ：磁盘文件保留完整记录，仅在模型输入中修剪 #### 4. 会话生命周期管理 重置模式 | 模式 | 行为 | | :&mdash;&mdash;- | :&mdash;&mdash;- | | daily | 固定时间重置（默认凌晨4点） | | idle | 闲置N分钟后重置 | | daily+idle | 取先到期者执行 | 会话记忆钩子 - 执行 /new 命令时自动保存上下文： 1. 提取最后15条消息 2.
