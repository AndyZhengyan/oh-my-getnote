原文： Claude 3.5首战复现21%顶会论文，人类博士无法取代，OpenAI：AI全是草台班子-36氪 🌟 研究背景与目标 - OpenAI发布全新框架PaperBench，旨在评估AI智能体复现顶尖研究的基础能力，目标直指AI智能体的科研潜力。若AI能完整复现ICML 2024 20篇优秀论文，意味着它要理解核心思想、开发代码库、运行实验并排除故障，这可是人类研究员都得花好几天的高难度任务呢！ 📄 PaperBench详情 - 论文选取 ：选取ICML 2024中20篇Spotlight和Oral论文，覆盖深度强化学习、鲁棒性和概率方法等12个主题。 - 评分标准 ：每篇论文都有详细评分标准，总计8316个可单独评估的评分项目，由论文原作者与OpenAI共同制定。采用层次结构，将复现目标分解为细粒度子任务。 - 任务要求 ：AI智能体收到论文和附录后，需提交包含复现所需代码的代码库，根目录要有reproduce.sh文件，能在新环境复现结果才算完成任务。 - 运行规则 ：智能体可上网，但不能用论文黑名单中网站资源，运行时间和计算能力无限制，开发者要提供必要API密钥，还有后处理监控工具检查是否违规。 ⚙️ 评估系统与成本 - OpenAI开发基于大模型的自动评判系统SimpleJudge，推出辅助工具JudgeEval验证其表现。对于PaperBench Code - Dev，每篇论文评判成本降至约10美元，比人工评分经济高效多啦！ 📈 实验结果 - 模型得分 ：对GPT - 4o、o1、o3 - mini、DeepSeek - R1、Claude 3.5 Sonnet（New）和Gemini 2.0 Flash等模型评估，Claude 3.5 Sonnet得分最高，达21.0%，o1得分为13.2%，其他模型低于10%。 - 失败模式 ：除Claude 3.5 Sonnet外，模型常提前结束任务，都未能制定有效策略，o3 - mini在工具使用方面有困难，显示当前模型在长期规划任务上有弱点。 - 迭代智能体 ：测试IterativeAgent，o1和o3 - mini得分显著提升，Claude 3.5 Sonnet在使用IterativeAgent时表现不如o1。 - 人类对比 ：招募8名ML博士挑战3篇论文子集，ML博士48小时内（3次尝试取最佳）拿下41.4%高分，o1仅26.6%，但在PaperBench Code - Dev中o1得分飙升至43.4%，显示AI在简化任务上有潜力。 🤔 结论与展望 - PaperBench揭示了AI智能体在科研中的潜力，也暴露了它们在长期规划任务中的局限性。未来随着模型能力提升，它们能否接近甚至超越41.4%的人类基准，真让人期待啊！
