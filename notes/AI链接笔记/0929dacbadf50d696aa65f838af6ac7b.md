原文： Nacos助力MCP生态发展：0代码适配与AI应用服务管理 🌟 MCP发展态势 - MCP发展速度惊人，今年2月Cursur、Winsurf、Cline引入MCP，近日OpenAI宣布支持，国内百度地图、高德地图发布MCP Server，还有众多相关服务供应商，生态愈发丰富成熟。 - 虽然AI短期面临ROI考验，但大家都不想错过这场“军备竞赛”，企业渴望提升MCP构建效率的方案。 💪 Nacos适配MCP Server优势 - “0改动”适配 ：Nacos作为MCP Registry，管理Tool元信息，将存量API转化成MCP协议，结合Higress AI网关，实现协议转换，让业务存量接口不改动就能生效MCP Server协议。 - “0代码”原理 ：对于用Nacos作注册配置中心的存量服务，增加服务接口信息即可实现模型调用上下文。引入“应用全局描述”改造服务，未注册服务手动注册，通过Higress插件暴露MCP协议。 - 转化原理 ：优先实现使用量高的Tool，借助Higress的SSE协议支持加速构建。通过Higress的MCP Server插件暴露Nacos管理的Tools，暴露tool/list接口返回Tool列表，进行协议转化将MCP协议的Json RPC转为普通HTTP请求并转发。 - 管理优势 ：存量API可快速构建MCP Server；MCP信息动态下发实时生效；管理MCP信息历史版本，方便对比和回滚；支持MCP信息灰度管理；提供密码配置加密；可进行MCP返回格式JSON转XML；具备MCP服务管理及健康检查能力。 🎬 演示复刻高德MCP Server - 环境准备 ：通过docker部署Nacos，安装docker和Kind，创建k8s集群，获取hgctl安装Higress，安装Kubectl并连接集群，部署redis，修改ConfigMap让Higress连接Nacos。 - 构建MCP Server ：申请高德开发者API KEY，在Nacos中配置高德API服务和获取出口IP服务，包括注册域名、创建配置文件描述接口信息。在Cursor中配置Higress作为MCP Server调用Tool，通过Agent模式查询天气，整个过程无需改造服务代码。 📈 未来展望 - 通过Nacos+Higress方案0代码改造连接Agent到存量应用，降低改造成本，后续可工具化简化接口配置。 - 解决Agent拉取Tool列表token消耗和模型性能下降问题，圈定有效Tool列表是演进方向。 - 支持MCP协议的Prompt、Resource等，助力MCP生态发展。 🚀 Nacos 3.0定位 - 架构已发布正式版本，定位AI应用服务管理平台，包含MCP动态管理，覆盖通用AI生态场景，帮助业务动态管理实时生效。 - 社区有多语言客户端生态，适配主流语言，助你低成本快速构建AI应用，未来会推出更多AI相关能力。
