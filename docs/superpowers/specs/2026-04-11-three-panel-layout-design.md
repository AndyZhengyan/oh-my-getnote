# 三栏布局重构设计文档

## 概述

将 Oh My Getnote 图谱页面从"底板+浮动卡片"结构改为类 ChatGPT 三栏独立面板布局，左侧面板可折叠、顶部有搜索触发按钮，弹出浮窗进行搜索。

## 整体结构

```
┌──────────────────────────────────────────────────────────────┐
│  Toolbar: Logo ···  统计数字 ··· 重置                       │ 52px
├──────────┬─────────────────────────────────┬────────────────┤
│ LeftNav  │         图谱主画布                │  RightPanel    │
│ (可折叠) │         ForceGraph               │  (可折叠)      │
│          │         (flex: 1)                │                │
│          │                                  │                │
└──────────┴─────────────────────────────────┴────────────────┘
```

三栏都是 flex 布局的一部分，随折叠/展开自动调整图谱区域大小。

---

## Toolbar（精简）

**高度**: 52px，`position: fixed` 固定顶部

**内容**:
- Logo + 标题
- 统计数字（655 篇 · 5240 条关联）
- 重置按钮

**去掉**: 领域下拉、类型下拉、搜索框

---

## LeftNav（左侧面板）

### 收起态（60px 宽）

只显示垂直排列的图标按钮，无文字标签：

| 图标 | 功能 |
|------|------|
| 搜索（放大镜） | 点击弹出搜索浮窗 |
| 知识领域列表 | 点击展开面板（不做过滤，直接展开） |
| 探索路径 | 点击展开面板 |
| 历史轨迹 | 点击展开面板 |

> 类似于 VS Code sidebar 的收起态，每个图标代表一个功能区。

### 展开态（280px 宽）

#### 顶部固定区（不受内容滚动影响）

- **搜索按钮**（放大镜图标 + "搜索"文字）：点击触发搜索浮窗弹出
- 收起/展开切换按钮（固定在右上角）

#### 下方内容区（可独立滚动）

**不激活搜索时**：
- 知识领域列表（NavItem，行为同现有逻辑）
- 探索路径 section（浏览轨迹，行为同现有逻辑）
- 历史轨迹 section（行为同现有逻辑）

**激活搜索时**（搜索浮窗已弹出，下方内容不变）：
- 浮窗覆盖在面板上方，内容区保持不变

---

## 搜索浮窗（SearchModal）

### 触发方式

点击 LeftNav 顶部的"搜索"按钮 → 浮窗从左面板右侧弹出，覆盖在面板上方。

### 浮窗内容

```
┌────────────────────────────────────┐
│  🔍 [搜索输入框........................] [领域▾] [类型▾]  │  ← 顶部行
├────────────────────────────────────┤
│  🔍 搜索结果（10 条）                 │
│  ┌────────────────────────────────┐ │
│  │ 笔记标题                        │ │
│  │ 标签 · 时间 · 领域              │ │
│  │ 摘要正文（2-3行）               │ │
│  └────────────────────────────────┘ │
│  ... 更多结果 ...                    │
└────────────────────────────────────┘
```

### 浮窗交互细节

| 交互 | 行为 |
|------|------|
| 输入关键词 | 实时触发搜索，300ms debounce，结果更新在下方面板 |
| 点击领域/类型筛选按钮 | 弹出下拉，在搜索结果中二次过滤 |
| 点击任意搜索结果 | RightPanel 展开显示笔记详情，浮窗关闭 |
| 点击浮窗外部 / ESC | 关闭浮窗 |
| 浮窗关闭后 | LeftNav 恢复展开态，内容区不变 |

### 搜索结果卡片内容

- 笔记标题（加粗，截断到 40 字）
- 标签 + 创建时间 + 领域（三者一行，灰色小字）
- 正文摘要（bodyPreview，最多 3 行，超出截断）

### 样式

- 宽度：360px（或比 LeftNav 宽 40px，覆盖在面板右侧）
- 最大高度：calc(100vh - 120px)，超出可滚动
- 背景：`rgba(255,255,255,0.96)` + `backdropFilter: blur(16px)`
- 边框：1px solid var(--border)，圆角 var(--radius-lg)
- 阴影：`0 8px 32px rgba(0,0,0,0.12)`

---

## 图谱主画布

- `flex: 1`，始终占满中央剩余空间
- 不再参与左右面板的"压缩"逻辑（折叠后自然扩展）
- ForceGraph 组件保持现有逻辑不变

---

## RightPanel（右侧面板）

### 收起态

- `width: 0`，不可见，无交互

### 展开态（380px 宽）

- 笔记正文（markdown 渲染）
- AI 摘要
- 推荐路径（multi-hop 结果）
- 关闭按钮

### 动画

- 宽度 0 ↔ 380px，0.25s ease-out
- 独立于 LeftNav，不联动

---

## 交互流程汇总

| 用户动作 | 结果 |
|---------|------|
| 点击 LeftNav 搜索图标 | 弹出搜索浮窗 |
| 输入搜索词 | 浮窗下方显示 Top 10 结果（标题+标签+时间+摘要） |
| 点击搜索结果 | RightPanel 展开（笔记详情），浮窗关闭 |
| 点击图谱节点 | RightPanel 展开（笔记详情） |
| 关闭 RightPanel | RightPanel 收起，图谱恢复 |
| 点击 LeftNav 折叠按钮 | LeftNav 60px ↔ 280px，图谱自动扩展/压缩 |
| LeftNav 收起时点击图标 | 展开对应功能区 |

---

## 实现顺序（建议）

1. **RightPanel 独立化**：确认 RightPanel 不再受 LeftNav 影响
2. **LeftNav 收起态改图标**：先把收起态的按钮文字去掉，只留图标
3. **Toolbar 精简**：去掉领域/类型下拉和搜索框
4. **搜索浮窗 SearchModal**：新建组件，实现搜索逻辑 + 浮窗 UI
5. **LeftNav 顶部搜索按钮**：绑定搜索浮窗触发
6. **左右折叠动画联调**：确保左右展开/收起流畅
